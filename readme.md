- We should aim to make these documents student AND instructor facing

- We should decide on heading levels. I think heading levels should err more towards making content readable on github, which to me looks like sections of the lesson being h2s. That lets the title be h1, and the pre-lesson stuff can be h3s to stand out. But h2's give us an underline to visually break up each section & make it more readable.
